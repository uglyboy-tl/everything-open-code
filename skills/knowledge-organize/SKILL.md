```bash
ls 收件箱/随手记/
```

若无文件或目录不存在，**立即报告并终止**。

**过滤规则**：标题为数字且 tags 含 `fleet` 的笔记为用户主动记录，**跳过整理**。

## 📋 笔记模板与规范

### 标准笔记结构

```markdown
---
date: YYYY-MM-DD HH:mm:ss
description: "简洁摘要（20字以内）"

# 项目主体属性（示例，无严格样式要求）
homepage: https://...          # 官网地址
docs: https://...              # 官方文档地址
github: owner/repo             # GitHub 仓库
docker: image:tag              # Docker 镜像
...
---

## 概览

简短描述（1-2 段），说明是什么、做什么。

## [要点分类]

### [具体要点描述]

具体内容。

## 参考资料

- [[相关笔记]] - 关联说明
```

**允许创建仅有概览的笔记**，用于主体概念的索引定位。

### 要点分类命名（二级标题）

- 应用工具 → 配置要点、使用要点、注意事项
- 理论概念 → 核心原理、关键定理、推导过程
- 技术方法 → 实践要点、实施步骤、常见问题
- 调试经验 → 排查要点、常见陷阱、诊断流程

### 具体要点命名（三级标题）

- ✅ 直接描述要点：`加载特定配置文件`、`缩进陷阱`
- ❌ 避免泛泛名称：`要点1`、`配置说明`

### 核心原则

- **独立主体概念**：每个笔记对应一个独立概念（类 Wiki）
- **不同概念分开**：即使是相关概念，只要独立就分开成两个笔记

## 🔄 整理流程

### 流程图

```
扫描收件箱 → 检索定位 → 判断归属 → 写入 → 更新索引 → 建立关联 → 清理
```

### Step 1: 扫描收件箱

```bash
ls 收件箱/随手记/
```

逐个处理文件。

### Step 2: 检索定位

读取快记内容，使用 **knowledge-retrieve** 检索知识库。

**决策分支**：
- >70% 匹配 → 合并到已有笔记
- 30%-70% 匹配 → 合并（新增章节）
- 未找到 → 创建新笔记

**禁止跳过检索直接新建笔记。**

### Step 3: 判断归属分类

```bash
cat 知识梳理/index.md
```

**关键词映射表**：

| 关键词示例 | 归属分类 |
|-----------|---------|
| Python, Git, Node.js, npm | 开发工具 |
| 数学, 概念, 定理, 算法 | 数学 |
| Debian, Linux, Shell | Debian |
| Docker, NAS, 容器 | NAS |
| Dotfiles, 配置管理 | Dotfiles |
| Hugo, 博客, 静态网站 | Hugo |
| Obsidian, 笔记, 知识管理 | Obsidian |
| OpenCode, opencode, AI | OpenCode |
| Selfhost, 自托管, 服务 | Selfhost |

无法确定归属时，创建 `待分类` 目录暂存。

### Step 4: 合并已有笔记

```bash
cat 知识梳理/<分类>/<笔记>.md
```

**合并策略**：
- 补充细节 → 追加到对应要点分类的子章节末尾
- 新增要点 → 在要点分类下新增子章节
- 矛盾信息 → 标注 `[待确认]`，禁止覆盖
- 缺少要点分类 → 先添加结构再合并

### Step 5: 创建新笔记

严格依从模板结构，允许仅有概览的索引笔记。

### Step 6: 建立关联

```bash
rg "关键词" 知识梳理/ -l
```

在笔记底部添加双链：
```markdown
## 参考资料

- [[相关笔记]] - 关联说明
```

### Step 7: 更新索引

```bash
cat 知识梳理/<分类>/index.md
```

在分类索引添加：
```markdown
- [[笔记名]] - 摘要说明
```

### Step 8: 清理收件箱

```bash
mv 收件箱/随手记/<文件>.md 收件箱/已整理/
```

禁止删除原文件，保留历史记录。

## 🛡️ 防御性约束

### ❌ 禁止行为

- 跳过检索定位直接新建笔记
- 跳过索引更新
- 删除收件箱原文件（必须移动）
- 偏离模板结构
- 覆盖矛盾信息（标注 `[待确认]`）

### ✅ 必须行为

- 先检索定位知识库
- 优先合并到已有笔记
- 依从模板结构
- 更新索引和关联
- 移动到已整理目录
- 保持双链格式一致：`[[笔记名]] - 说明`
