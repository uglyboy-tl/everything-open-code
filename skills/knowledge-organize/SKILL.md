# 知识整理

## 整理流程

```
扫描收件箱 → 检索定位 → 判断归属 → 规范化 → 写入 → 更新索引 → 建立关联 → 清理
```

## Step 1: 扫描收件箱

```bash
ls 收件箱/随手记/
```

列出待整理文件，逐个处理。

## Step 2: 检索定位

读取快记内容，使用 **knowledge-retrieve** 检索：

**决策分支**：
- 找到高度相关笔记（>70%匹配） → 合并到已有笔记
- 找到弱相关笔记（30%-70%匹配） → 合并（新增章节）
- 未找到相关笔记 → 新建笔记

## Step 3: 判断归属

自动判断分类：

```bash
cat 知识梳理/index.md
```

根据关键词确定分类：
- Python/TypeScript/Git → 开发工具
- 数学/概念 → 数学
- Debian/Linux → Debian
- Docker/NAS → NAS

## Step 4: 合并已有笔记

```bash
cat 知识梳理/<分类>/<笔记>.md
```

**合并策略**：
- 补充细节 → 追加到现有章节末尾
- 新增子主题 → 在合适位置新增章节
- 矛盾信息 → 标注 `[待确认]` 不覆盖

**更新**：
- 在合适位置插入新内容
- 更新 frontmatter 的 `lastmod`

## Step 5: 创建新笔记

**Frontmatter**：
```yaml
---
date: YYYY-MM-DD HH:mm:ss
lastmod: YYYY-MM-DD HH:mm:ss
description: "简洁摘要"
tags: [标签1, 标签2]  # 可选
url: https://...      # 可选
---
```

**内容组织**：
- 按主题自然展开，无固定模板
- 标题直接描述主题或功能
- 底部添加"参考资料"区块

**命名规范**：
- 使用中文名称，简洁明了
- 禁止特殊字符：`/ \ : * ? " < > |`
- 用连字符替代空格

**颗粒度原则**：
- 一条笔记一个主题概念
- 不拆分细碎子概念（避免碎片化）
- 内容过多时才拆分子分类

## Step 6: 建立关联

```bash
rg "关键词" 知识梳理/ -l
```

在笔记底部添加双链：
```markdown
## 参考资料

- [[相关笔记]] - 关联说明
```

## Step 7: 更新索引

```bash
cat 知识梳理/<分类>/index.md
```

在分类索引添加：
```markdown
- [[笔记名]] - 摘要说明
```

## Step 8: 清理收件箱

```bash
mv 收件箱/随手记/<文件>.md 收件箱/已整理/
```

移动到已整理目录，保留历史记录。

## 防御性约束

**禁止**：
- ❌ 整理前跳过检索定位
- ❌ 创建碎片化笔记
- ❌ 跳过索引更新
- ❌ 删除收件箱原文件（应移动）

**必须**：
- ✅ 检索定位优先
- ✅ 优先合并到已有笔记
- ✅ 新建笔记添加完整 frontmatter
- ✅ 自动更新索引和关联
- ✅ 移动到已整理目录
