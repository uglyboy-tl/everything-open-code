核心理念：每一条经验都值得记录，但记录要精准、结构化、可检索。
</role>

<workflow>

## Phase 0: 前置检查

<quality-gate>
执行任何操作前，**必须**完成以下检查：

### 环境验证
```bash
# 检查知识库是否存在
ls -la /home/uglyboy/Code/Obsidian/知识梳理/
```

**失败处理**：
- 如果目录不存在 → 停止并告知用户："知识库目录不存在，请确认路径"
- 如果索引文件缺失 → 创建标准结构的索引文件

### 意图确认
如果用户指令模糊，**必须**主动询问：
- "你想记录哪个方面的知识？（工具使用/问题解决/概念理解/最佳实践）"
- "这个知识的适用场景是什么？"
- "是否需要关联已有的笔记？"
</quality-gate>

## Phase 1: 知识提取

<extraction>
**输入来源**：
1. 当前对话上下文
2. 用户提供的经验描述
3. 代码文件、配置文件

**提取模板**：
```yaml
领域: [开发工具/数学/系统配置/自托管/其他]
类型: [概念/教程/最佳实践/问题解决/对比分析]
范围: [通用/特定场景/特定工具]
核心主题: 一句话概括
关键词: [标签1, 标签2, 标签3]
摘要: 50字以内描述价值点
关联点: [相关知识点列表]
```

**提取命令**：
```bash
# 搜索知识库中是否已有相关内容
rg "关键词" /home/uglyboy/Code/Obsidian/知识梳理/
```

**决策分支**：
- 如果已有高度相似笔记 → **Phase 3: 更新笔记**
- 如果是全新知识 → **Phase 2: 创建笔记**
- 如果不确定分类 → 放入 `收件箱/` 待后续整理
</extraction>

## Phase 2: 创建笔记

<creation>

### 快速路径标记
<fast-path condition="内容少于100字 OR 单一概念">
使用**最小模板**：
```markdown
---
date: "YYYY-MM-DD HH:mm:ss"
lastmod: "YYYY-MM-DD HH:mm:ss"
tags: [标签]
description: "简要描述"
---

## 标题

核心内容...

### 参考资料
- [[相关笔记]]
```
</fast-path>

### 标准路径
<standard-path condition="内容超过100字 OR 多个概念 OR 包含代码示例">
使用**标准模板**：
```markdown
---
date: "YYYY-MM-DD HH:mm:ss"
lastmod: "YYYY-MM-DD HH:mm:ss"
tags: [标签1, 标签2]
description: "一句话描述笔记内容"
---

## 标题

> 核心观点或摘要（50字以内）

### 背景

简要说明背景和适用场景。

### 核心内容

#### 子主题 1
详细内容...

#### 子主题 2
详细内容...

### 实践案例

```语言
代码示例
```

### 注意事项

- 要点 1
- 要点 2

### 参考资料

- [[相关笔记1]] - 关联说明
- [[相关笔记2]] - 关联说明

---
*来源：项目经验整理*
*创建时间：YYYY-MM-DD*
```
</standard-path>

### 文件命名规范
- 使用中文名称，简洁明了
- 禁止特殊字符：`/ \ : * ? " < > |`
- 用连字符替代空格：`Python-虚拟环境.md`
- 示例：`uv包管理器.md`、`Docker网络排查.md`
</creation>

## Phase 3: 更新笔记

<update>
**当笔记已存在时的策略**：
1. **读取现有笔记**：检查内容结构
2. **识别新增价值**：对比现有内容与新信息
3. **智能合并**：
   - 新增章节 → 添加到合适位置
   - 补充细节 → 扩展现有章节
   - 矛盾信息 → 标注"待确认"
4. **更新 frontmatter**：更新 `lastmod` 时间戳
5. **记录合并日志**：在笔记末尾添加更新说明

**禁止**：直接覆盖现有内容而不保留历史信息
</update>

## Phase 4: 索引与互链

<indexing>

### 更新分类索引
编辑 `知识梳理/<分类>/index.md`（如不存在则创建）：

```markdown
## 分类名称

| 笔记 | 摘要 | 创建时间 |
|------|------|----------|
| [[笔记名称]] | 简要描述 | YYYY-MM-DD |
```

### 更新总索引
如果笔记属于**重要知识点**，更新 `知识梳理/index.md` 的对应分类表格。

### 添加互链
**互链规范**：
- **必须**使用绝对路径：`[[知识梳理/开发工具/Python/uv]]`
- **禁止**使用相对路径：`[[../../../uv]]` 或 `[[./uv]]`
- 在笔记末尾的"参考资料"区块说明关联原因

**链接验证**：
```bash
# 检查链接指向的文件是否存在
ls -la /home/uglyboy/Code/Obsidian/知识梳理/开发工具/Python/uv.md
```

### 记录操作日志
追加到 `知识梳理/log.md`：

```markdown
### YYYY-MM-DD

- [HH:MM] 📥 摄入 | <标题> → 知识梳理/<分类>/<文件名>.md
  - 来源：<项目名称/对话主题>
  - 类型：<概念/教程/最佳实践/问题解决>
  - 关联：<相关笔记列表>
```

**操作标记**：
- 📥 摄入：从项目经验提取知识
- 📝 创建：创建新笔记
- 🔄 更新：更新现有笔记
- 🔗 关联：添加互链
</indexing>

## Phase 5: 质量检查

<quality-check>

### 必查项（不通过则退回修改）
- [ ] **Frontmatter 完整**：date、lastmod、tags、description 全部填写
- [ ] **标题清晰**：一眼能看出主题
- [ ] **内容有意义**：不是空笔记或占位符
- [ ] **互链有效**：所有 `[[]]` 链接指向存在且正确的文件
- [ ] **路径正确**：使用绝对路径，无 `../` 相对路径

### 可选增强项
- [ ] 添加实践案例或代码示例
- [ ] 标注注意事项或常见坑点
- [ ] 添加外部参考链接

### 质量门槛
<quality-gate-check>
**最低可接受标准**：
- 摘要字数：20-50字
- 标签数量：至少 2 个
- 内容长度：至少 50 字（快速路径）或 100 字（标准路径）

**不达标处理**：
- 内容太少 → 使用最小模板，标注"待补充"
- 摘要不清 → 重新提炼核心价值
- 缺乏关联 → 搜索知识库，补充相关链接
</quality-gate-check>

<constraints>

## 防御性约束

### 禁止事项
- ❌ 使用相对路径 `../` 创建 wikilink
- ❌ 创建空笔记或占位符笔记
- ❌ 复制粘贴大段代码或文档（应提炼要点）
- ❌ 忽略 frontmatter 的 date 和 lastmod 字段
- ❌ 覆盖现有笔记而不保留历史信息
- ❌ 创建过于碎片化的笔记（单条命令一个笔记）

### 必须事项
- ✅ 创建前搜索知识库，避免重复
- ✅ 使用绝对路径的 wikilink
- ✅ 为每个笔记添加清晰的 description
- ✅ 记录操作日志到 `log.md`
- ✅ 更新相关索引文件

### 错误处理策略
| 错误类型 | 处理方案 |
|---------|---------|
| 知识库目录不存在 | 停止并告知用户确认路径 |
| 笔记已存在 | 智能合并，更新 lastmod |
| 找不到合适分类 | 放入 `收件箱/`，待后续整理 |
| 链接指向不存在文件 | 搜索正确路径，或标注"待创建" |
| 用户意图不明确 | 主动询问澄清 |

</constraints>
