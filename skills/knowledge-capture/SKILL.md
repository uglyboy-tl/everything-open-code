## 快记流程

```
发现经验 → 触发判断 → 检索去重 → 写入收件箱
```

## Step 1: 触发判断

**必须满足**：
- ✅ 已验证：亲身尝试并确认有效
- ✅ 通用性：可迁移到其他项目或场景
- ✅ 非显而易见：文档未明确写出
- ✅ 具体可执行：明确的操作/配置/方法

**立即拒绝**：
- ❌ 未验证的"可能有效"
- ❌ 仅适用于当前项目的路径、依赖
- ❌ 标准文档中的明显事实

## Step 2: 检索去重

使用 **knowledge-retrieve** 技能检索相关知识。

**处理策略**：
- 完全重复 → 不记录，告知用户已存在
- 部分重复 → 精简待记录内容，只保留新增部分
- 相关但不重复 → 记录并在"参考链接"中建立关联
- 未找到 → 全部记录

## Step 3: 快速记录

```markdown
# 标题

## 发现
- 要点 1
- 要点 2

## 价值说明
为什么重要？避免了什么坑？

## 错误尝试（可选）
- ❌ 尝试 A → 失败原因

## 标签
#标签1 #标签2

---
来源: 项目/对话
```

## Step 4: 保存位置

```bash
收件箱/随手记/<标题>.md
```

文件名直接使用标题，避免特殊字符。

## 值得记录的内容

**调试发现**：
- 错误信息误导时的真实原因
- 问题定位的关键线索
- 通用排查流程

**工具特性**：
- 未写入文档的命令/参数
- 与官方文档不一致的实际行为
- 工具的 workaround

**配置经验**：
- 非显而易见的配置依赖
- 多个配置项的隐式关系
- 用户提出的额外工作规范

**架构决策**：
- 选择方案的关键因素
- 放弃其他方案的原因

**外部资源**：
- 发现的优质网络资料

---

## 不记录的内容

- 具体项目的文件路径
- 未验证的知识点
- 标准语言/框架行为
- 知识库已有类似内容

---

## 快速判断

```
发现经验
  ├─ 已验证? NO → ❌ 拒绝
  ├─ 通用性? NO → ❌ 仅当前项目
  ├─ 非显而易见? YES → ❌ 标准知识
  └─ 知识库已有? YES → ❌ 重复
  └─ NO → ✅ 记录
```
